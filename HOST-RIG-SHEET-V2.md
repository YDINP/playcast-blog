> 관련: [HOST-RIG-PARTS.md](./HOST-RIG-PARTS.md) (파츠 목록/우선순위)

## 1. 왜 다시 설계하는가 — 실제로 터진 3가지

| # | 문제 | 원인 | 결과 |
|---|---|---|---|
| 1 | 눈 위에 옅은 색 패치가 보임 | 눈 파츠에 **눈꺼풀 살색이 딸려옴** | base 피부톤과 미묘하게 어긋남 |
| 2 | 파츠 위치·크기를 손으로 맞춰야 함 | 파츠가 **캔버스에서 떨어져 나온 조각** | 좌표 소실 → 상대좌표 역산 + 눈대중 보정 |
| 3 | 크로마키가 캐릭터를 갉아먹을 위험 | 배경색이 **캐릭터 팔레트와 충돌** | 민트 머리를 초록 배경으로 키잉하면 머리가 뚫림 |

핵심 통찰 하나로 요약하면:

> **파츠는 "오려낸 조각"이 아니라, 같은 캔버스 위에 위치를 유지한 채 그려진 "레이어"여야 한다.**

Live2D/Spine의 소스 PSD가 정확히 그 형태입니다. 레이어를 껐다 켜면 그 자리에 그대로 나타나죠.
생성 모델에게도 똑같이 요구하면 됩니다 — **격자 타일 하나하나가 "전체 캔버스"이고, 그 안에 딱 한 레이어만 그려져 있는 시트.**

---

## 2. 설계 원칙 5가지

### 원칙 1 — 타일 = 전체 캔버스 (제자리 레이어)
파츠를 모아 놓지 말고, **모든 타일을 같은 크기의 전체 캔버스**로 만들고 그 안에 해당 레이어만 그린다.
캐릭터의 위치·크기·각도는 모든 타일에서 완전히 동일. 나머지 영역은 전부 배경색.

```
┌──────────┬──────────┬──────────┐
│ ① base   │ ② 눈흰자  │ ③ 홍채   │   ← 각 칸이 전부 같은 전체 캔버스.
│ (민얼굴)  │  (제자리) │ (제자리)  │      해당 레이어만 그려져 있고
├──────────┼──────────┼──────────┤      나머지는 배경색.
│ ④ 눈꺼풀  │ ⑤ 눈썹    │ ⑥ 입     │   → 잘라내면 곧바로 겹쳐지는 레이어가 된다.
└──────────┴──────────┴──────────┘
```

이렇게 하면 내가 할 일은 "타일을 격자로 자르고 배경만 지우기"뿐입니다. **위치·크기 추정이 완전히 사라집니다.**

### 원칙 2 — 파츠는 자기 것만 담는다 (피부 금지)
눈 파츠에 살색이 들어가면 base 피부와 어긋납니다. 그러니:

- **눈흰자 레이어**: 흰자(공막)만. 눈꺼풀 살색·속눈썹 없음.
- **홍채 레이어**: 홍채+동공+하이라이트만. 흰자 없음.
- **눈꺼풀/속눈썹 레이어**: 선화(속눈썹, 쌍꺼풀 라인, 눈매 그림자)만. **피부는 base가 담당.**
- **입 레이어**: 입술·구강·이·혀만. 주변 피부 없음.
- **눈썹 레이어**: 눈썹 털만.

경계가 애매한 것(눈꺼풀 그림자 등)은 **base 쪽에 그린다**. base는 한 장뿐이라 색이 어긋날 일이 없다.

### 원칙 3 — 가려지는 부분까지 그린다
레이어를 움직이면 가려졌던 영역이 드러납니다.

- **눈흰자**: 눈꺼풀에 가려질 위쪽까지 완전한 아몬드 형태로. (반쯤 감을 때 필요)
- **홍채**: 완전한 원/타원으로. 위아래가 눈꺼풀에 잘리지 않은 상태로. (시선 이동 시 필요)
- **머리(head)**: 앞머리에 가려질 이마까지. (고개 갸웃 시 필요)
- **몸통(body)**: 머리에 가려질 목/어깨 윗부분까지.

### 원칙 4 — 배경은 **투명(알파)이 최선**, 크로마는 폴백
**OpenAI 이미지 모델(gpt-image)은 알파를 직접 출력할 수 있습니다** — API에서 `background: "transparent"` +
`output_format: "png"`. 이게 가능하면 크로마키 자체가 필요 없고, 아래 문제들이 전부 사라집니다.

- 키잉 임계값 튜닝 불필요
- 경계에 남는 배경색 번짐(디스필) 불필요 — 반투명 안티에일리어싱이 그대로 보존됨
- 배경색이 캐릭터 팔레트를 갉아먹을 위험 0

⚠️ 단, **Higgsfield MCP의 `gpt_image_2`는 이 파라미터를 노출하지 않습니다**(파라미터가 `resolution`/`quality`뿐).
투명 배경을 쓰려면 **OpenAI API를 직접 호출**해야 합니다(사용자 API 키 필요).

**폴백 — 크로마 배경을 써야 한다면** 배경색은 **캐릭터 팔레트에 없는 색**이어야 합니다.
로지의 팔레트: 민트그린 · 라벤더/퍼플 · 살구빛 피부 · 검정 후드 · 빨간 마이크 · 청록 눈

| 후보 | 판정 |
|---|---|
| 초록 #00FF00 | ❌ **민트 머리와 충돌** → 머리가 뚫린다 |
| 마젠타 #FF00FF | ❌ 라벤더 머리와 근접 → 경계 오염 |
| 파랑 #0000FF | ⚠️ 청록 눈과 근접 |
| **노랑 #FFF200** | ✅ **팔레트에 없음.** c.png/c1~c4에서 검증 완료 |

판정식: `노랑도 = min(R,G) − B`. 노랑 배경은 240 이상, 캐릭터 색은 전부 60 이하로 갈립니다
(피부 #FFE0C0조차 32 수준이라 여유가 큼).

> Higgsfield `remove_background`는 시트처럼 **조각이 여러 개** 흩어진 이미지에서는 피사체 하나만 남기고
> 나머지를 지울 위험이 커서 부적합합니다.

### 원칙 5 — z-order를 미리 확정한다
합성 순서가 곧 리그 구조입니다.

```
body → head(민얼굴) → 눈흰자 → 홍채 → 눈꺼풀/속눈썹 → 눈썹 → 볼홍조 → 입 → 옆머리 → 앞머리 → 헤드셋/마이크
```

- 홍채가 **눈꺼풀보다 아래**여야 시선을 움직여도 눈꺼풀 밖으로 안 나갑니다(자동 클리핑).
- 앞머리가 **가장 위**여야 고개를 움직일 때 머리카락이 얼굴 위로 자연스럽게 덮입니다.
- 헤드셋은 앞머리 위 — 실제로 착용한 물건이니까요.

## 2-B. 눈 구조 — 홍채가 움직일 "여유"를 설계에 넣어야 한다

지금 시트의 눈은 **홍채가 눈을 거의 꽉 채웁니다.** 이대로 잘라 쓰면 홍채를 움직일 공간이 없습니다
(실제로 현재 리그의 시선 추적 이동폭이 ±8px에 불과한 이유가 이것입니다).

```
지금 (여유 없음)                 필요한 것 (여유 있음)
 ┌───────────────┐               ┌───────────────┐
 │▓▓▓▓▓▓▓▓▓▓▓▓▓│  홍채가 흰자를   │░░▓▓▓▓▓▓▓▓░░│  좌우로 흰자가
 │▓▓▓▓▓▓▓▓▓▓▓▓▓│  꽉 채움         │░░▓▓▓▓▓▓▓▓░░│  보이는 여백
 └───────────────┘  → 이동 불가     └───────────────┘  → ±15~20% 이동 가능
```

**설계 규칙 3가지**

1. **흰자(eye_white)는 완전한 아몬드 형태**로, 눈꺼풀에 잘리지 않은 상태로 그린다.
   반쯤 감긴 눈을 만들 때 위쪽이 필요하고, 홍채가 움직일 때 좌우가 필요하다.
2. **홍채(iris)는 완전한 타원**으로, 눈꺼풀·흰자에 잘리지 않은 상태로 그린다.
   그리고 **홍채 폭이 흰자 폭의 약 65~70%** 가 되게 그린다 → 좌우로 각각 15% 정도의 이동 여유가 생긴다.
   (애니 스타일 특성상 이보다 작게 그리면 캐릭터 인상이 바뀌므로 70% 근처가 상한선)
3. **눈꺼풀/속눈썹 레이어가 홍채보다 위**에 온다 → 홍채가 위아래로 움직여도 **눈꺼풀이 자동으로 잘라준다**.
   별도의 마스크 작업이 필요 없다. (현재 리그는 흰자 알파를 CSS 마스크로 써서 이걸 흉내내고 있음)

> 즉 "홍채가 안 움직인다"는 문제는 코드가 아니라 **그림 단계에서 해결**해야 합니다.
> 흰자를 크게, 홍채를 그보다 작게, 그리고 둘 다 잘리지 않은 완전한 형태로.

## 2-C. 머리 / 몸 분리 — 목에서 자르고, 이음새는 옷깃·머리카락이 가린다

고개를 갸웃하려면 **head 레이어가 body와 분리되어 회전**해야 합니다. 문제는 "자른 자리"입니다.

```
       ┌──────────┐
       │  head    │  ← 회전하는 레이어 (얼굴+두상+귀). 목 윗부분까지 포함
       │  ╲    ╱  │
   ────┼───╳────┼────  ← 절단선: 턱 아래, 옷깃 위 (Y = 목 중간)
       │  body    │  ← 고정 레이어. 머리에 가려질 목 전체를 그려둔다
       └──────────┘
```

**규칙 4가지**

1. **절단선은 "턱 아래 ~ 옷깃 위"의 목 중간**에 둔다.
2. **body는 목을 완전하게 그린다** — head에 가려질 부분까지. head가 기울면 그 뒤가 드러나기 때문.
   (이걸 안 하면 고개를 5°만 갸웃해도 목에 구멍이 뚫린다)
3. **head도 목 윗동강을 조금 물고 내려온다**(겹침 마진 ~10px). 겹쳐야 회전 시 틈이 안 생긴다.
4. **회전 피벗은 목 밑동**(절단선보다 살짝 아래, 쇄골 중앙 위쪽). 여기가 실제 목뼈 위치라 자연스럽다.

**이음새가 안 보이는 이유**: 로지는 ① 후드 칼라가 목을 감싸고 ② 옆머리가 목 양옆을 덮습니다.
그래서 회전각을 **±6° 이내**로 제한하면 절단선이 노출되지 않습니다. (Live2D 모델도 대부분 이 방식)

> 부수 효과: head를 분리하면 **호흡·끄덕임도 머리만** 움직일 수 있어 훨씬 살아 있는 느낌이 납니다.
> 지금은 몸통까지 통째로 흔들려서 "그림이 흔들린다"에 가깝습니다.

## 2-D. 업계 표준은 어떻게 나누나 (Live2D 공식 + 모델러 가이드 조사)

[Live2D 공식 PSD 가이드](https://docs.live2d.com/4.2/en/cubism-editor-tutorials/psd/)와
[모델러 커뮤니티 가이드](https://community.live2d.com/discussion/318/modeling-101-layering-the-character)를
확인한 결과, 우리가 세운 원칙과 **대부분 일치**했고 **놓친 것 3가지**가 있었습니다.

### 일치한 것 ✅
- **"가려지는 부분까지 그린다"** — 공식 문구: *"눈동자에 가려지는 부분도 보이게 되므로 흰자를 반드시 그려라"*
- **흰자를 클리핑 마스크로 쓴다** — 홍채가 흰자 밖으로 안 나가게. (지금 우리가 CSS `mask-image`로 하는 것과 동일한 발상)
- **목은 크게 그린다** — 공식 문구: *"얼굴이 움직일 때 잘려 보이지 않도록 목을 크게 그릴 것"*
- **머리카락은 앞머리 / 옆머리 / 뒷머리로 분리**, 뒷머리는 얼굴 뒤

### 놓쳤던 것 ⚠️

**① 눈 하이라이트(eye gleam)는 별도 레이어**
표준 눈 스택(위→아래): `눈꺼풀 주름 → 위 속눈썹 → 아래 속눈썹 → 눈꺼풀 → 하이라이트 → 홍채 → 흰자 → 얼굴`
하이라이트를 홍채에서 분리하면, **홍채가 움직여도 하이라이트는 제자리**에 둘 수 있습니다(빛 방향이 고정되니 훨씬 자연스럽다).
지금 우리 홍채 파츠는 하이라이트가 박혀 있어서 눈동자를 굴리면 반사광까지 같이 굴러갑니다.

**② 깜빡임은 "선화 3단"이 아니라 "살색 눈꺼풀이 내려오는" 구조**
표준은 **불투명한 살색 눈꺼풀 레이어**가 눈알 위를 덮으며 내려옵니다(변형/이동). 눈알 자체는 그대로 있고요.
다만 이건 **메시 변형이 가능한 Live2D 기준**입니다. 우리는 CSS 레이어라 변형이 안 되므로,
**눈꺼풀 3단(뜬/반쯤/감은)을 각각 그려 교체**하는 방식이 맞습니다. 대신 눈꺼풀은 **살색 불투명**으로
그려야 아래 눈알을 확실히 가립니다(선화만 그리면 눈알이 비쳐 보임 — 우리가 흰자 패치에서 겪은 문제와 같은 종류).

**③ 입은 "구강 + 살색 테두리"가 필요**
공식 문구: *"입 안쪽은 원본보다 한 치수 크게 그리고, 삐져나오지 않도록 살색으로 채워라."*
즉 입 레이어는 `입술 + 구강(크게) + 이/혀`이고, **입술 바깥에 살색 여백**을 둬서 벌어진 입이 얼굴 밖으로 새지 않게 합니다.
우리 입 파츠엔 이 살색 테두리가 없어서, 큰 입("아")이 턱선을 침범할 위험이 있습니다.

### 그래서 최종 z-order (우리 CSS 리그용으로 번역)

```
body → head(민얼굴) → 뒷머리
     → 흰자 → 홍채(이동) → 하이라이트(고정)
     → 눈꺼풀(살색 불투명: 뜬/반쯤/감은 중 하나)
     → 속눈썹(선화) → 눈썹 → 볼홍조
     → 입(구강+살색테두리)
     → 옆머리 → 앞머리 → 헤드셋·마이크
```

## 3. 시트 구성 (3장으로 나눠 요청)

한 장에 12칸을 넣으면 칸당 해상도가 부족합니다(2K 기준 칸당 ~600px). **3장으로 쪼갭니다.**

### 시트 A — 신체 (6칸)
| 칸 | 레이어 |
|---|---|
| 1 | **body**: 목 아래 몸통 + 후드(칼라 포함). ★목을 **턱 아래까지 크게** 그림(머리가 기울면 드러남) |
| 2 | **head**: 두상 + 민얼굴(눈·눈썹·입·홍조·속눈썹 전무, 코 점만) + 귀 + 목 윗동강(겹침 마진) |
| 3 | **hair_back**: 뒷머리 (얼굴 뒤로 가는 부분) |
| 4 | **hair_side**: 옆머리 좌우 |
| 5 | **hair_bangs**: 앞머리 |

### 4-1. 마스터 프리앰블 (모든 시트 앞에 붙임)

```
너는 Live2D/Spine 리깅용 2D 캐릭터 레이어 시트를 그리는 일러스트레이터다.
지금 만드는 것은 "완성된 그림"이 아니라 "레이어 소스"다.

[캐릭터] "로지" — 애니메 스타일. 민트그린→라벤더 그라데이션 단발 보브컷,
큰 청록색 눈, 검은 후드(민트색 끈), 고양이 귀 달린 검은 게이밍 헤드셋 + 빨간 마이크 붐.
정면 상반신(가슴 위). 부드러운 셀 셰이딩, 깔끔한 선화.

[가장 중요한 규칙 — 제자리 레이어]
- 출력은 격자 시트다. 각 칸은 "캐릭터 전체 캔버스"이며, 캐릭터의 위치·크기·각도는 모든 칸에서 완전히 동일하다.
- 각 칸에는 지정된 레이어 하나만 그리고, 나머지는 전부 배경색으로 비운다.
- 레이어를 그 자리에서 그려라. 조각을 가운데로 모으거나 확대하지 마라. 잘라내지도 마라.
- 즉, 모든 칸을 겹쳐 놓으면 완성된 캐릭터가 되어야 한다.

[배경]
- (알파 지원 시) 완전 투명 배경. PNG.
- (알파 불가 시) 순수 노랑 #FFF200 단색. 그림자·글로우·그라데이션 금지.
  이 색은 캐릭터 팔레트에 없으므로 배경 제거용이다. 캐릭터 어디에도 이 노랑을 쓰지 마라.

[가려지는 부분]
- 다른 레이어에 가려질 부분도 완전한 형태로 그려라.
  (예: 흰자는 눈꺼풀에 안 잘린 완전한 아몬드, 홍채는 잘리지 않은 완전한 타원,
   머리는 앞머리에 가려질 이마까지, 몸통은 머리에 가려질 목까지)

[금지] 텍스트·라벨·번호·테두리·프레임·워터마크·드롭섀도우.
```

**OpenAI API로 투명 배경을 뽑을 때** (권장):

```bash
curl https://api.openai.com/v1/images/generations \
  -H "Authorization: Bearer $OPENAI_API_KEY" -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-image-1",
    "prompt": "<마스터 프리앰블 + 시트 프롬프트>",
    "size": "1536x1024",
    "background": "transparent",
    "output_format": "png",
    "quality": "high"
  }'
```
`background: "transparent"` 가 핵심입니다. 이러면 크로마키 단계 자체가 사라집니다.

## 5. 받은 뒤 파이프라인 (내가 하는 일)

1. **격자 분할** — 시트를 균등 분할(칸 수를 알고 있으므로 좌표 계산만).
2. **노랑 키잉** — `min(R,G) − B > 임계` 소프트 알파 + 디스필. (기존 스크립트 재사용)
3. **정렬 검증** — 각 레이어의 캐릭터 위치가 같은지 확인. base 대비 어긋나면 머리 bbox로 보정.
   ✅ 제자리 레이어라 원래는 보정이 필요 없어야 정상. 어긋나면 그 시트는 **재생성 대상**.
4. **z-order 합성** → 리그에 배선:
   - 시선 추적 = 홍채 레이어를 ±px 이동 (눈꺼풀 레이어가 위에 있어 자동 클리핑)
   - 깜빡임 = lid_open → lid_half → lid_closed 전환
   - 표정 = 눈썹 4종 × 눈꺼풀 3종 조합
   - 립싱크 = 입 7종을 음소별로 전환 (TTS 붙이면 그대로 음소 립싱크)
   - 고개 갸웃 = head + hair를 body와 분리해 회전
5. **검증** — 합성 결과를 확대 렌더해 이음새·색 어긋남 확인.
