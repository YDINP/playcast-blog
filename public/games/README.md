# public/games/ — 게임 이미지 넣는 곳

- `public/games/*.jpg` — 글의 **대표 이미지(썸네일/OG)**
- `public/games/scenes/*.jpg` — **씬 배경**. 씬플레이어가 장면마다 깔아주는 배경.

## 현재 채워진 것 (2026-07-13)

스팀 상점 페이지의 **공식 스크린샷**을 스팀 공식 API(`store.steampowered.com/api/appdetails`)로 받아
1280px / JPEG q78로 재인코딩해 넣었습니다(합계 약 1.3MB).

- **가이드·버튜버 편(4글)** — 특정 게임이 없어 그라디언트 배경 유지.

## 넣는 방법 (수동)
1. 게임사 **공식 프레스킷 / 스토어 페이지**에서 이미지를 받습니다.
2. 대표 이미지는 `public/games/<이름>.jpg`, 씬 배경은 `public/games/scenes/<이름>.jpg`로 저장.
3. 글 frontmatter `thumbnail` / 씬의 `image:` 에 경로를 씁니다. 권장 규격 **가로 16:9(1280×720 이상)**.
4. 파일이 없으면 자동으로 **그라디언트 플레이스홀더**가 뜹니다(깨진 이미지 안 뜸).
   단, `thumbnail`에 없는 파일을 적어두면 **404가 납니다** — 파일을 넣기 전엔 주석 처리할 것.

## 저작권 (⚠️ 2026-07-13 정책 완화)
기존 기준은 "글당 대표 1장(부종성)"이었으나, **씬마다 공식 스크린샷을 쓰는 쪽으로 완화**했습니다.
그만큼 인용 근거는 약해지므로 아래를 반드시 유지합니다.

- [x] **공식 출처만**: 스팀 상점/프레스킷 스크린샷. 유저 촬영본·팬아트·유출본 금지
- [x] **출처표기**: frontmatter `imageCredit` (시청 페이지에 자동 노출)
- [x] **광고 미표시**: 공식 이미지를 쓰는 글은 `ads: false`
- [x] **본문 우위**: 이미지는 배경, 콘텐츠의 본체는 호스트 대사(트랜스크립트)
- [ ] **가이드라인 준수**: 각 게임사 콘텐츠 이용 약관의 허용 범위 확인
- [ ] **takedown 대응**: 권리사 요청 시 즉시 교체/삭제 (교체 지점 = 이 폴더 + 글의 `image:`)

> ⚠️ 로고·박스아트를 채널 브랜딩으로 쓰지 말 것. 소개 맥락의 식별용 삽입만.
> ⚠️ 사이트 전체 수익화 단계에선 AI 창작 배경으로 교체를 권장.
