# PRD — VIP 시청 페이지 레이아웃 개편 + 호스트 부스 분리

> 2026-07-13 · 대상: `src/pages/watch/[...slug].astro`, `src/components/HostStage.astro`, `public/js/scene-player.js`, `src/styles/global.css`

## 문제

1. **호스트가 영상 안에 갇혀 있음** — `.sp-host`가 `.sp-stage` 내부 우하단 오버레이(absolute)라 배경을 가리고, 캐릭터가 "영상의 일부"로 읽힌다. 채널의 정체성(버츄얼 호스트)이 약하게 드러남.
2. **시청 페이지가 유튜브 클론** — 구독 버튼 / 채널 행 / 우측 추천 사이드바 등 유튜브 UI를 그대로 베껴 차별성이 없고, 실제로는 존재하지 않는 기능(구독)까지 흉내낸다.

## 목표

- 호스트를 영상 프레임 **밖 우측 전용 부스 패널**로 분리 (방송 스튜디오 느낌). 립싱크/표정/`is-talking` 동작은 그대로 유지.
- 시청 페이지를 유튜브 문법 대신 **에디토리얼/스트리밍 스타일**로 재구성.

## 설계

### 1. 스테이지 = 영상 + 부스 (`.sp-shell`)

```
.sp-shell (grid: 1fr / 300px)
├── .sp-stage    배경 크로스페이드 + 자막 + 컨트롤 (호스트 없음)
└── .sp-booth    LIVE 뱃지 · 호스트(.sp-host) · 이름/역할 · 말하는 중 이퀄라이저
```

- `HostStage`에 `booth` prop 추가. **홈 히어로는 기존 인라인 오버레이 유지**(`booth=false` 기본값) → 히어로의 몰입감은 그대로, 시청 페이지만 분리형.
- `scene-player.js`는 `.sp-stage` 안에서 호스트를 찾으므로, **shell 범위까지 확장 탐색**으로 수정 (`stage.closest('.sp-shell')`).
- 반응형: 900px 이하에서 부스가 영상 아래 가로 스트립으로 전환.

### 2. 시청 페이지 구조 (유튜브 요소 제거)

| 제거 | 대체 |
|---|---|
| 채널 행 + 구독 버튼 | 호스트 크레딧이 **부스 패널**로 흡수. 가짜 구독 버튼 삭제 |
| 우측 추천 사이드바 | **하단 풀폭 추천 그리드** (본문 폭을 넓게 사용) |
| 설명란 "...더보기" 접기 | 상단 히어로 헤더의 리드 문단으로 승격 (SEO상 항상 노출) |
| 대본 = 밋밋한 버튼 목록 | **챕터 칩 레일(가로 스크롤) + 대사 타임라인** |

최종 순서: 히어로 헤더(카테고리·제목·리드·메타) → 스테이지+부스 → 액션바(좋아요/공유/길이) → 챕터 레일 → 대본 타임라인 → 추천 그리드 → 댓글.

## 제약

- **SEO 회귀 금지**: 대사 전문(트랜스크립트)은 계속 정적 DOM에 노출. `display:none` 금지.
- `[data-transcript="{id}"] .tr-item` 셀렉터와 **아이템 순서 = 씬 인덱스** 계약 유지 (scene-player.js가 이 순서로 점프).
- `.watch`/`.watch-side` 스타일은 홈(`index.astro`)이 사용 중 → 삭제하지 말 것.
