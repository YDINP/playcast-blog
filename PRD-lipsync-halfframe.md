# PRD — 립싱크 개폐 중간 프레임(Half-open) 추가

> 작성일 2026-07-22 · 대상: playcast-blog(VIP) 씬플레이어 호스트 립싱크

## 배경 / 문제

현재 호스트(로지) 립싱크는 **다문 입(mouth-rest) ↔ 완전 개구 5모음(아·에·이·오·우)** 사이를
`data-viseme` 하드 토글 + opacity 크로스페이드(0.11s)로만 잇는다. 중간 개폐 프레임이 없어
말 시작/끝·자모 전환 시 입이 "뚝" 스냅되는 느낌이 남는다.

## 목표 (이번 작업 = 옵션 A 단독)

말할 때 입이 **닫힘 → 반개(half) → 개구**로 단계지게 지나가도록 **중간 입 프레임 1장**을 추가해
개폐 모션을 부드럽게 한다. (옵션 B=자음 비젬 확장은 이번 범위 아님)

## 최우선 제약 — 규격/벡터 불변 (사용자 명시)

1. **해상도 동일**: 500×500 (기존 char2 애셋 전부 500² VP8X webp).
2. **규격 동일**: webp, 정사각 500², 알파 포함.
3. **벡터(오버레이 정렬)값 동일**: CSS `left:0; top:0; width:100%` + 입 클립박스
   `clip-path: inset(36% 47% 56% 47%)` 를 **그대로 재사용**. 신규 프레임도 동일 좌표계.
4. **얼굴 불변 보장**: 신규 `lip-half`는 `base.webp`와 입 박스 밖 영역이 **픽셀 동일**해야 한다.
   → 생성물에서 **입 박스 영역만** 추출해 base 위에 합성(sharp)하여 나머지 얼굴 드리프트 원천 차단.

## 접근

### 애셋
- `public/host/char2/lip-half.webp` (500×500) 신규 1장.
- 생성: `lip-a.webp`(정렬된 개구 컷) 기준 i2i 편집 → "동일 얼굴, 입만 반개".
- 정규화(sharp): 500×500 강제 + **입 박스 픽셀만** base.webp에 합성 → 얼굴 나머지 = base와 동일.
- 검증: base 위 오버레이 → 입 클립박스 안에서 이음매/정렬 스샷 확인(Playwright).
- 실패(드리프트 심함) 시 폴백: sharp로 lip-a 입 영역을 세로 압축해 반개 근사(AI 미사용, 완전 결정론).

### 코드
- `HostRig.astro`: `<img class="rig-part rig-lip l-half" src="/host/char2/lip-half.webp">` 추가.
- `scene-player.js`: `_loop`에서 개폐 경계(닫힘↔모음)로 바뀔 때 목표 직전 `half`를 짧게 경유.
  - `this._lastVis` 추적. 모음 진입 & 직전 closed → `half` 세팅 후 micro-timeout(≈MOUTH_MS/2)에 목표 승격(가드).
  - closed 진입 & 직전 모음 → `half` 경유 후 closed.
  - 그 외(모음→모음)는 기존처럼 직접(이미 크로스페이드로 부드러움).
- `global.css`: `.sp-host.is-talking[data-viseme="half"] .rig-lip.l-half { opacity:1 }` (클립박스 동일).

## 검증 기준(완료 정의)

- [ ] lip-half.webp = 500×500 webp, 입 박스 밖 = base 픽셀 동일(합성으로 보장).
- [ ] 오버레이 CSS(정렬/클립) 기존값 그대로, 신규 좌표/트랜스폼 도입 없음.
- [ ] 재생 시 입이 닫힘→반개→개구로 지나감(스샷/영상 확인), 이음매 없음.
- [ ] `astro build` 통과.

## 리스크

- **정렬 드리프트 = 입 이음매** → 입 박스만 base에 합성으로 차단(최대 리스크 방어).
- 글자당 ~130ms라 half가 너무 빨라 안 보일 수 있음 → micro-timeout/전환 타이밍 튜닝.
- webp 장수 +1(6→7장, 용량 미미).
