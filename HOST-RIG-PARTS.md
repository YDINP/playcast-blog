## 2. 제작 규칙 (이게 안 지켜지면 못 씁니다)

1. **모든 파츠는 같은 캔버스에, 같은 위치로.** 파츠만 그리고 나머지는 투명. **절대 잘라내지(crop) 말 것.**
   - 캔버스: **1448 × 1086** (지금 받은 c1~c4와 동일). 캐릭터 위치·크기도 동일.
   - 예: `pupil_L.png`는 1448×1086 캔버스에 **왼쪽 눈동자만** 그려져 있고 나머지는 전부 투명.
2. **배경**: 투명(알파) PNG가 최선. 투명이 안 되면 지금처럼 **순수 노랑 `#FFF200`** 크로마로 주세요(내가 마스킹합니다). 노랑은 캐릭터 색에 안 쓰이므로 안전합니다.
3. **동일 캐릭터·동일 조명·동일 선화 두께.** 파츠 간 색/그림자 톤이 어긋나면 합성 시 이질감이 납니다.
4. **가려지는 부분도 그려주세요.** 예: `head`의 민얼굴은 앞머리에 가려질 이마까지 그리고, `eye_white`는 눈꺼풀에 가려질 위쪽까지 그립니다. 레이어를 움직이면 가려졌던 부분이 드러납니다.
5. **좌/우는 파일을 나눠주세요**(`_L`, `_R`). 한 장에 둘 다 있으면 눈동자를 따로 못 움직입니다.
6. 시트로 줄 거면 **2그리드(가로 2칸)** 형식 유지 — 지금 파이프라인이 자동으로 쪼갭니다.

## 3. GPT에 던질 프롬프트

### 3-1. 마스터 프리앰블 (항상 앞에 붙이기)

```
너는 버츄얼 유튜버용 2D 캐릭터 리깅 에셋을 만드는 일러스트레이터다.

[캐릭터] "로지" — 애니메 스타일, 민트~라벤더 그라데이션 단발 보브컷, 청록색 큰 눈,
검은 후드(민트색 끈), 고양이 귀 달린 검은 게이밍 헤드셋 + 빨간 마이크 붐.
정면 상반신(가슴 위까지). 부드러운 셀 셰이딩, 깔끔한 선화.

[출력 규격 — 반드시 지킬 것]
- 캔버스: 1448 x 1086 픽셀 고정. 캐릭터의 위치·크기·각도는 모든 이미지에서 완전히 동일.
- 배경: 순수 노랑 #FFF200 단색 (다른 노랑 금지). 그림자/글로우도 배경에 넣지 말 것.
- 요청한 부위만 다르게 하고, 나머지는 픽셀 단위로 동일하게 유지.
- 캐릭터를 잘라내거나 확대/축소/이동하지 말 것.
- 텍스트, 워터마크, 프레임, 테두리선 넣지 말 것.
```

### 3-2. 시트별 프롬프트 (하나씩 복붙)

**① 눈동자 방향 시트 (시선 추적용)** — 가장 중요
```
위 규격대로, 가로 2칸 시트를 만들어라 (1448x1086 안에 좌/우 두 컷).
좌: 눈동자가 화면 왼쪽 끝을 보는 로지 (얼굴·표정·머리·몸은 기본 상태 그대로, 눈동자 위치만 변경)
우: 눈동자가 화면 오른쪽 끝을 보는 로지
눈꺼풀·눈썹·입·머리카락은 기본 컷과 완전히 동일해야 한다.
```
같은 식으로 **위/아래 시선** 시트도 한 장 더 요청.

**② 눈꺼풀 단계 시트 (자연스러운 깜빡임)**
```
위 규격대로, 가로 2칸 시트:
좌: 눈을 반쯤 감은 로지 (눈꺼풀이 눈동자 절반을 덮음, 졸린 표정 아님 — 중립)
우: 눈을 완전히 감은 로지 (편안한 곡선, 미소 없음 — 중립)
눈썹·입·머리·몸은 기본 컷과 완전히 동일.
```

**③ 눈썹 시트**
```
위 규격대로, 가로 2칸 시트:
좌: 눈썹만 크게 치켜올린 로지 (놀람) — 눈·입은 기본 그대로
우: 눈썹만 찌푸린 로지 (화남/집중) — 눈·입은 기본 그대로
```
```
위 규격대로, 가로 2칸 시트:
좌: 눈썹 안쪽 끝이 처진 로지 (슬픔/걱정) — 눈·입은 기본 그대로
우: 눈썹이 중립인 로지 (기준 컷)
```

**④ 모음 입모양 시트 (TTS 립싱크)** — 2장으로 나눠 요청
```
위 규격대로, 가로 2칸 시트:
좌: 입을 크게 벌린 "아" 발음 (턱이 내려감)
우: 입을 옆으로 벌린 "이" 발음 (이가 살짝 보임)
눈·눈썹·머리·몸은 기본 컷과 완전히 동일.
```
```
위 규격대로, 가로 2칸 시트:
좌: 입을 동그랗게 오므린 "우" 발음
우: 입을 살짝 벌린 "에" 발음
```
(+ "오" 발음과 "닫힌 입"으로 한 장 더)

**⑤ 팔 동작 시트 (P2)**
```
위 규격대로, 가로 2칸 시트:
좌: 오른손을 들어 반갑게 흔드는 로지 (얼굴·머리·몸통 위치는 기본과 동일, 팔만 다름)
우: 오른손 검지로 화면 오른쪽을 가리키는 로지
```

### 3-3. 투명 배경 레이어를 직접 뽑을 수 있는 툴을 쓴다면 (권장)

GPT 대신 **레이어 분리가 되는 툴**(Clip Studio / Photoshop / Krita)에서 아래처럼 내보내주시면 마스킹 없이 바로 씁니다.

```
1448x1086 캔버스, 각 레이어를 개별 PNG(투명 배경)로 내보내기:
body, head, hair_front, hair_side_L, hair_side_R,
eye_white_L, eye_white_R, pupil_L, pupil_R,
eyelid_L_open/half/closed, eyelid_R_open/half/closed,
brow_L_neutral/up/down/sad, brow_R_neutral/up/down/sad,
mouth_closed/smile/a/i/u/e/o, blush, headset, mic
※ 모든 레이어는 같은 캔버스 좌표에, 가려지는 부분까지 그린 상태로.
```

## 4. 받은 다음 내가 하는 일 (파이프라인)

1. 노랑 크로마 마스킹 + 소프트 알파 + 디스필 (`.tmp/build-expr.mjs` 계열 스크립트 재사용)
2. 머리 bbox로 컷 간 정렬 오차 보정 (지금까지 ≤2.5px)
3. 웹용 축소(600px 기준) + PNG 재인코딩
4. 리그 레이어 순서로 합성:
   `body → head → eye_white → pupil → eyelid → brow → blush → mouth → hair_side → hair_front → headset/mic → fx`
5. `scene-player.js` 리그 엔진에 배선 (시선 추적, 눈꺼풀 3단 깜빡임, 음소 립싱크, 머리 관성)

## 5. 주의 (지금까지 실제로 터진 함정)

- **파츠 경계가 그리려는 부위를 지나가면 안 된다.** 얼굴 밴드 상단이 눈썹을 가로질러 페더링에 눈썹이 반투명해지고, 아래 기본 눈썹이 비쳐 **눈썹이 두 개로 보였다**.
- **컷마다 프레이밍이 바뀌면 못 쓴다.** 같은 캐릭터를 매번 새로 생성하면 머리 위치가 흔들리는데, 지금까지는 운 좋게 오차 2.5px 이내였다.
- **리그 박스 비율이 이미지 비율(600:911)과 어긋나면** 눈·입·표정이 전부 얼굴에서 밀린다(`aspect-ratio` 유지 필수).
